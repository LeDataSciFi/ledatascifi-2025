Classification models

  • A classification model is when you're predicting an outcome with discrete levels and is best divided into two parts:
    1. the statistical problem of learning a model to predict, ideally, class probabilities;
    2. the decision problem to take concrete action based on those probability predictions.
  • Our current example:
    1. Step 1: Estimate via logit the probability of a loan charge off based on application data X.
    2. The fitted logit model produces, for a given vector of data X, the predicted probability that y=1 (charge off) for those values of X.
    3. This is not a classification, its a probability, a number in [0,1].
    4. Step 2: A classification model that uses logit takes the logit predicted probability, and outputs a prediction: YES, or NO.
  • The considerations we learn in this application here apply to many settings: Predicting what stocks to buy/sell is a two part problem as well
    1. Estimate $f()$ to model a stock's future return: $ret_{t+1}=f(X_t)$
    2. Apply $f()$ to the latest data to generate predicted returns for all stocks, and then pick which stocks to buy and/or sell.

Evaluating the classification model

First, we cover the classic "ML" model evaluation outputs. These aren't made for business settings, but are useful.

  1. Calibration display
  2. Classification report
  3. Confusion Matrix
  4. Precision-Recall Curve: Can your model have high recall and precision if we adjust the threshold?
  5. ROC Curve: Can your model have low false positives and high recall if we adjust the threshold?

Imbalanced data makes classification harder

  • It makes the second step (turn logit predicted probs into binary predictions) hard
  • Options:
    1. Manually pick a different threshold. If the predicted probablity is above X%, predict the outcome is positive. The default is 50%.
    2. Change estimator to "weight" groups equally (class-weight=balanced in tree models and logit... equiv to "oversampling")
    3. Remove data (meaning: undersample overrepresented group)
    4. Add data (meaning: oversample underrepresented group)
